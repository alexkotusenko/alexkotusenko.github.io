The Caesar Cipher

One of the simplest and widely known ciphers is the Caesar Cipher. Just like its name suggests, it was named after Julius Caesar, who reportedly used it to encrypt military-related messages.

Before explaining how this cryptosystem works, I should touch upon the significance of indexes of letters in an alphabet. As we all know, the English alphabet consists of 26 letters. Each one has its own index, representing its position in the alphabet. By this logic, the index of the letter A is 1 and the index of the letter X is 24.

Indexes of letters are commonly used in cryptography to label letters using numbers, which make mathematical operations possible.

Many ciphers use the index property of letters to encrypt messages and the Caesar Cipher is one of them.

To encrypt a message using the Caesar Cipher one needs to:

  1. Define the key value
  2. Get the index of each letter in the message
  3. Add the key value to the index of each letter to obtain the new index of every letter *
  4. Convert the newly obtained indexes into letters, thus obtaining the ciphertext

* Note: if the new index is bigger than 26 or smaller than 0, an additional operation has to be performed.

The key used in the Caesar Cipher is an integer between 1 and 26 inclusively.

As an example, I will explain how to encrypt “Hello World!” using a key value of 5.

For this demonstration, I will use a conversion table to majorly simplify the task. It is also highly recommended to use a table just like this one when performing manual encryption and decryption.

First of all, we need to define the key value. As mentioned already, for this example it’s 5.

Now, we need to get the index value of each letter in the message.

Note: In this case, we will not take into account the indexes of symbols such as exclamation points or question marks as well as smaces, as they will not be encrypted. Also, the capital letters will stay capital after conversion. Please also note that these details can vary from one case to another, depending on what the parties agreed on.

The following table displays the index of each letter in the message.

H E L L O W O R L D
8 5 12 12 15 23 15 18 12 4

Now, we have to add 5 to every single index to obtain a list of new indexes.

13 10 17 17 20 28 20 23 17 9

Now, as you can see, one of the newly obtained indexes is equal to 28. Whenever the value of a new index is more than 26, you need to divide 26 from the new index until it’s between 1 and 26 inclusively. Likewise, whenever your new index is smaller than 26, you need to add 26 to the new index until it’s between 1 and 26 inclusively. The reasoning behind that is that there is no letter with the index smaller than 1 or bigger than 26 to associate the new index to. You can think of this index system as a clock, where whenever it strikes past twelve, the arrow just loops around the clock, thus always staying inside of this limited loop.

Keeping that in mind, we can say that the new index of the sixth letter is actually 2 (28-26). Now, the last thing we have to do is to translate the newly obtained indexes into letters, thus compiling the ciphertext.

13 10 17 17 20 2 20 23 17 9
M J Q Q T B T W Q I

As you can see, the ciphertext of “Hello World!” encrypted using a key value of 5 is “Mjqqt Btwqi!”.

In the case of the Caesar Cipher, the decryption process is similar to the encryption one. To decrypt a message, one has to subtract the key value from the index of each letter, thus getting the original index of a letter and compiling the plaintext. It is also required to make sure that the index values obtained while decrypting a message are between 1 and 26 inclusively using the operation mentioned above, otherwise it wouldn’t be possible to associate the indexes in question with the corresponding letters of the alphabet.
