The Affine Cipher and Introduction to Modular Arithmetic II

Despite its rather simple encryption method, the Affine Cipher does not work the same way when it comes to decryption.

To decrypt a message encypted using this cipher, one has to use the following function:

D(y) ≡ a-1 (y - b) mod 26

In this situation, the D(y) variable represents the index of the plaintext version of the letter in question, while the y variable represents its ciphertext version’s index. This equation can only be used to decrypt the ciphertext (the only variable one can replace in order to solve it (besides a-1 and b) is y).

Also, as you can see in the equation, the a value is not the same as in the encryption function. Instead, it’s been replaced by its multiplicative inverse. See, within modular arithmetic, one is always working with integers, and decimals are never used. As a result, modular arithmetic does not allow for division, since it often results in a decimal number. Thus, instead of dividing a number x by a number y, we multiply x by y’s multiplicative inverse (y-1).

The multiplicative inverse of a number can be calculated using the following formula:

a ✕ a-1 ≡ 1 mod 26

Note: the multiplicative inverse of a given integer varies depending on the modulo you’re working with.

As it happens, there’s only so many possible a values, since not every integer within modulo 26 has a multiplicative inverse. A given integer can only have a multiplicative inverse within mod x if 1 is the only positive integer divisor of x and said integer. For example, the multiplicative inverse of 4 within mod 26 does not exist, since the smallest divisor of 4 and 26 is 2, since both numbers can be divided by it and result in a positive integer. As a result, there’s only 12 integers that have a multiplicative inverse within mod 26, and that’s why there’s only 12 possible a values.
