The Brute Force Attack

Since the dawn of cryptography, there has been an endless race between people who created ciphers and the ones who were trying to crack them. In fact, such a race is still very much present in our times. Cipher breakers constantly find new ways to break a cryptosystem, which gives cipher creators an insight on the weaker parts of the cipher in question. In the history of cryptography, more often than not, ciphers that were deemed to be unbreakable for several decades or even centuries got cracked after a while, thus drastically changing our understanding of the properties of ciphers.

One of the most ancient and simplest ways to crack a cryptosystem is done by trying every single combination of the key in question, which is called The Brute Force Attack. One might think that this way of cracking a cipher roots in cryptography, but that's not the case. This code breaking attack can also be applied to many different things in life, even the ones not entirely related to cryptography. Imagine a code lock, where the key is a four digit number. If brute force were to be applied in bypassing the lock, one would need to try every single key combination possible within the system in question. In the case with a four digit number code, one would have to try up to ten thousand different keys to successfully bypass the lock!

As you might have guessed already, the brute force approach is not always the best way to crack a cipher. However, it allows us to cross out the ciphers that are vulnerable to it, and that’s the case of the Caesar Cipher.

As mentioned before, the key within the Caesar Cipher is a number between 1 and 26 inclusively. Therefore, there are 26 possible key variations for this cipher. Unlike the code lock example mentioned above, where there are thousands of possible code combinations, given that this cipher only has a few possible keys, it becomes vulnerable to such an attack.

For this demonstration, I will show how vulnerable this cipher is by using the brute force to decrypt the following message: “Lzak ak s kwujwl ewkksyw.”

+1 Mabl bl t lxvkxm fxlltzx.
+2 Nbcm cm u mywlyn gymmuay.
+3 Ocdn dn v nzxmzo hznnvbz.
+4 Pdeo eo w oaynap iaoowca.
+5 Qefp fp x pbzobq jbppxdb.
+6 Rfgq gq y qcapcr kcqqyec.
+7 Sghr hr z rdbqds ldrrzfd.
+8 This is a secret message.

After trying 8 key combinations out of 26 possible ones, we stumble upon a readable text, which implies that the message has successfully been encrypted.

Although this cipher was deemed to be strong when it was used for the first time by Julius Caesar, after some time, some cryptographer managed to crack it, which allowed cipher makers to analyze the weak spots in a cipher and improve the security of the other ones they created.
